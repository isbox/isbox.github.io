webpack配置知多少

发表于 2020-04-22

记录一些webpack中不常用的配置

resolve

  • symlinks

    默认值: true

    官方解释：是否将符号链接(symlink)解析到它们的符号链接位置(symlink location)
    启用时，符号链接(symlink)的资源，将解析为其_真实_路径，而不是其符号链接(symlink)位置。注意，当使用符号链接 package 包工具时（如 npm link），可能会导致模块解析失败。

    重点在第二句话, 会导致npm link解析失败

    在开发组件时通常会在组件中使用 npm link, 此时会有两个步骤

    关于prefix: 在大多数系统上是 /usr/local, 在Windows上为 %AppData%\npm
    官方解释

    1. 在全局文件夹 {prefix}/lib/node_modules/<package> 创建符号链接(symlink), 链接到npm link执行命令的包
    2. 将所有的 bins 链接到 bins in the package to {prefix}/bin/{name}

    然后在项目中使用 npm link xxx 将全局安装目标链接到项目的node_modules文件夹中

    也就是说webpack无法正确的解析node_module中的symlink导致引入模块失败

    使用link调试时需要设置 symlinks: false 来禁止webpack解析符号链接
