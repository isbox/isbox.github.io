【读书】你不知道的js(上) — 1. 作用域

发表于 2020-11-23

javascript常被归为“动态”或“解释执行”语言，但实际上是一门编译语言（与传统编译语言不同）

传统编译流程主要分为三步骤：

  1. 分词/词法分析（Tokenizing/Lexing）：

    此步骤会将字符组成的字符串分解成有意义的代码块，这些代码块被称为词法单元（token）

  2. 解析/语法分析（Parsing）：

    此步骤会将词法单元流（数组）转化为一个由元素逐级嵌套所组成的程序语法结构树。这个树被称为抽象语法（Abstract Syntax Tree，AST）

  3. 代码生成

    此步骤会将AST转化为可执行代码，以及创建变量并为其分配内存等

对于javascript来说，大部分编译都发生在代码将要执行的前几微秒，并不是在构建时

javascript编译

  1. 引擎，负责整个javascript程序的编译和执行过程
  2. 编译器，负责语法分析以及代码生成
  3. 作用域，负责收集并维护由所有的声明的标识符（变量）组成的一系列查询，并实施一套非常严格的规则，确定当前执行的代码对这些标识符的访问权限

分析var a = 2;

  1. 编译器会询问作用域是否有一个变量a存在于同一作用域集合中，如果存在，编译器会忽略，否则创建一个新的变量，命名为a
  2. 编译器会为引擎生成运行时所需要的代码来处理a = 2的赋值操作，引擎会首先询问作用域，在当前作用域中是否存在变量a，如果存在，引擎会使用改变量，否则引擎会继续查找改变量直到找到，将2赋值给改变量。如果没有找到，引擎会抛出一个异常

总结：在赋值操作中会执行两个步骤，声明变量（如果没有声明过），赋值该变量

LHS和RHS查询，“赋值操作的目标是谁(LHS)”以及“谁是赋值操作的源头(RHS)，并非赋值的左右侧”

词法作用域

作用域共有两种主要的工作模型，1是词法作用域（大多数语言），2是动态作用域（Bash 脚本）等

  • 词法阶段

    大部分标准语言编译器的第一个工作阶段叫作词法化(也叫单词化)，会对源代码字符串进行检查，而词法作用域就是在定义在词法阶段的作用域

    即词法作用域是由写代码时将变量和块作用域写在哪里来决定的

  • 作用域的遮蔽效应

    作用域查找会逐级向上查找，直到找到第一个匹配的标识符时停止。在多层嵌套作用域中可以定义同名标识符，这些标识符会“遮蔽”掉外部的同名标识符。

    全局变量会转为全局对象，可以不通过全局词法名称而是通过访问全局对象对其进行访问。

  • 无论函数在哪里被调用，也无论它如何被调用，它的词法作用域都只由函数被声明时所处 的位置决定

    词法作用域只会查找一级标识符，如果使用了foo.bar.baz这种，词法作用域只会尝试查找foo标识符，找到后对象访问器会接管bar和baz属性的访问

with (现在不推荐使用)

with 通常被当作重复引用同一个对象中的多个属性的快捷方式

a出现在全局作用域上因为在with中对a赋值时，o2对象上并不存在a属性，触发的LHS查询会继续向上查询，知道全局作用于也没找到，于是在全局作用域上创建a并赋值，此时，变量a被泄露到全局上了

尽管 with 块可以将一个对象处理为词法作用域，但是这个块内部正常的 var 声明并不会被限制在这个块的作用域中，而是被添加到 with 所处的函数作用域中。

不推荐在代码中使用eval与with函数，因为eval与with会在运行时修改/创建新的作用域，而js引擎会在编译阶段进行数项性能优化，而有的优化是依赖于代码词法的静态分析，预先确定所有变量和函数定义的位置，才能在执行过程中快速找到标识符

但如果引擎在代码中发现了 eval(..) 或 with，它只能简单地假设关于标识符位置的判断 都是无效的，因为无法在词法分析阶段明确知道 eval(..) 会接收到什么代码，也无法知道传递给 with 用来创建新词法作用域的对象的内容到底是什么

最悲观情况是引擎完全不会做优化

函数作用域与块作用域

函数作用域的含义是指，属于这个函数的全部变量都可以在整个函数的范围内使用及复用

也可以使用反向思维：从所写的代码中挑选出一个任意的片段，然后用函数声明对它进行包装，实际 上就是把这些代码“隐藏”起来了

函数表达式与函数声明

区分函数声明和表达式最简单的方法是看 function 关键字出现在声明中的位置(不仅仅是一行代码，而是整个声明中的位置)，如果 function 是声明中 的第一个词，那么就是一个函数声明，否则就是一个函数表达式

块作用域

在使用var定义变量时，它写在哪里都是一样的，因为它们最终都会属于外部作用域。

let关键字可以将变量绑定到所在的任意作用域中(通常是 { .. } 内部)。换句话说，let为其声明的变量隐式地绑定在了所在的块作用域。

块作用域也可以显示创建

块作用域在垃圾回收上非常有用
