类型

最新的 ECMAScript 标准定义了 8 种数据类型：

其中 7 种原始类型：boolean，string，number，null，undefined，bigInt，symbol

还有一种类型：object

原始类型本身并不是对象

简单基本类型（string、boolean、bigint、symbol、number、null 和 undefined）本身并不是对象

实际上“JavaScript 中万物皆是对象”，的说法是错误的。

typeof null 会返回 object 从而被当作对象实际上是语言本身的bug

原理是这样的，不同的对象在底层都表示为二进制，在 JavaScript 中二进制前三位都为 0 的话会被判断为 object 类型，null 的二进制表示是全 0，自然前三位也是 0，所以执行 typeof 时会返回“object”

内置对象

javascript中有些子对象类型，通常称为内置对象，有些名字看起来和原始类型一样

  • String
  • Number
  • Boolean
  • Object
  • Function
  • Array
  • Symbol
  • BigInt
  • DateRegExp
  • Error

在 javascript 中，它们实际上只是一些内置函数，有些函数可以当作构造函数使用（new 构造调用）从而构造一个对应子类型的对象

对于 Object、Array、Function 和 RegExp(正则表达式)来说，无论使用文字形式还是构造形式，它们都是对象

对于 null 和 undefined 没有对应的构造形式，它们只有文字形式

属性描述符

从 ES5 开始，所有的属性都具备了属性描述符

myObject 对象属性 a 对应的属性描述符（也被称为“数据描述符”），除了value值表示对应属性的值，还有3个特性

  • writable（可写）
  • enumerable（可枚举）
  • configurable（可配置）

在创建普通属性时属性描述符会使用默认值，也可以使用 Object.defineProperty 来添加一个新属性或者修改一个已有属性

在语言规范中，obj.a 在 obj 上实现了 [[Get]] 操作（类似调用 [[Get]] 函数 ），对象中默认的内置 [[Get]] 操作会首先寻找对象中是否有相同名字的属性，如果找到则返回属性值，如果在当前对象中没有找到，会延原型链向上寻找。当无论如何都找不到同名属性时，会返回undefined

属性访问与访问作用域变量不同，前者没找到会返回undefined，后者则是抛出一个 ReferenceError 异常

有获取属性的 [[Get]] 就有对应的 [[Put]]

[[Put]] 行为的触发取决于许多因素，如果已经存在这个属性

  1. 属性是否是访问描述符？如果是并且存在 setter 就调用 setter
  2. 属性的数据描述符中writable是否是false？如果是，在非严格模式下静默失败，在严格模式下抛出 TypeError 异常
  3. 如果都不是，将该值设置为属性的值

Getter / Setter

ES5中可以通过 getter 和 setter 部分改写默认操作（只能用于单个属性上），getter 是一个隐藏函数，会在获取属性值时调用。setter 也是一个隐藏函数，会在设置属性值时调用

当一个属性定义了getter、setter，或两者都有时，这个属性会被定义为 “访问描述符”，对于访问描述符来说，javascript会忽略其 value 和 writable 特性

不管是对象文字语法 get a() {} 还是defineProperty显示定义，都会在对象中创建不包含值的属性，对于属性的访问会自动调用隐藏函数getter，函数返回值则为属性属性访问返回值
