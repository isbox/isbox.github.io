原型链

Object.prototype

所有普通的 [[Prototype]] 链的尽头，最终都会指向内置的 Object.prototype，所以大多普通对象都会包含如：toString，toValue，hasOwnProperty方法

属性设置和屏蔽

当对对象中的属性进行修改/新增时，会经历一些过程，并不仅仅是赋值

如：myObj.foo = 123;

  • 当 foo属性存在于myObj对象上时，这条赋值语句只会修改已有属性的值

  • 如果foo不存在于myObj对象上时，[[Prototype]]会被遍历，如果在原型上仍找不到，foo属性会被添加到myObj上并赋值

  • 如果foo不存在于myObj对象上且在[[Prototype]]上被找到且writable: true，foo属性会被添加到myObj上，且发挥屏蔽，之后再访问该属性会直接使用myObj中的foo属性，不在向上寻找

  • 如果foo不存在于myObj对象上且在[[Prototype]]上被找到且writable: false，则无法修改/在myObj上创建新属性，严格模式下则会发生报错

    ( Tips: 使用Object.defineProperty不会收到限制 )

  • 如果foo不存在于myObj对象上且在[[Prototype]]上被找到且为一个setter，则只会调用该setter

某些情况下会发生隐式屏蔽

类

JavaScript 和面向类的语言不同，它并没有类来作为对象的抽象模式或者说蓝图，JavaScript 中只有对象

“类”函数

javascript模仿类使用了函数的一种特殊特，所有的函数默认都会拥有一个名为 prototype 的公有并且不可枚举

但是在 JavaScript 中，并没有类似的复制机制。你不能创建一个类的多个实例，只能创建多个对象，它们 [[Prototype]] 关联的是同一个对象，这个机制通常被称为原型继承

构造函数

关键字 new，在面向类的语言中构造类实例时也会用到它。看起来我们执行了类的构造函数方法，调用方式很像初始化类时类构造函数的调用方式

普通的函数调用前面加上 new 关键字之后，new 会劫持所有普通函数并用构造对象的形式来调用它

在 JavaScript 中对于“构造函数”最准确的解释是，所有带 new 的函数调用

Class

ES6 新语法糖 class 除了语法更好看，还解决了如下问题

  1. 不再引用杂乱的 prototype
  2. 继承可以直接使用 extends，不需要通过 Object.create 替换 prototype
  3. 可以使用 super 直接实现多态
  4. extends 可以更自然的拓展对象，甚至是内置对象，如 Array 等

class陷阱

class并非引入了“类”机制，class 其实只是 prototype（委托） 机制的一种语法糖

也就是说，class 并不会像传统面向类语言一样，在声明时复制所有行为。如果修改了父类，那所有的子类和所有实例都会受到影响

class也存在意外屏蔽的问题

super 也会存在一些小问题，由于 javascript 的 class 实际是通过 prototype 模拟，调用 super 实际是调用 prototype 链上同名方法，当 prototype 链上方法被修改后，调用的 super 应该也被修改，实际却并不是这样的

但是出于性能考虑，super的绑定是“静态”的，它会在声明时“静态”绑定
