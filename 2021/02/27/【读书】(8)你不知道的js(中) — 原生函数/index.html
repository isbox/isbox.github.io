原生函数作为构造函数

数组（Array），对象（Object），函数（Function），正则表达式（RegExp），一般都会使用常量形式来创建，实际上，通过常量

和使用构造函数形式创建的效果都是一样的（创建的值都会通过封装对象包装）

如果非必要，应尽量避免使用构造函数形式创建，会有意想不到的结果

Date(..) 和 Error(..)

Date

创建日期对象需要使用 new Date()，单独调用 Date()，会返回当前日期字符串

Error

Error 同样可以不使用 new 关键字调用，创建错误对象主要是为了获得当前运行栈的上下文（大部分 javascript 引擎可以使用.stack 来访问）

栈上下文信息包括函数调用栈信息和产生错误的代码行号，通常 Error 会与 throw 一起使用

除 Error 之外，还有一些针对特定错误类型的原生构造函数，如 EvalError、 RangeError、ReferenceError、SyntaxError、 TypeError 和 URIError，这些构造函数很少被直接使用，它们会在程序异常时自动被调用（如使用未声明的变量产生 ReferenceError 错误）

Symbol

使用符号命名对象属性不容易导致重名

符号可以最为属性名，但是无论代码还是控制台都无法查看和访问其值

ES6 中一些预定符号，以 Symbol 的静态属性出现，如 Symbol.create，Symbol.iterator，Symbol.asyncIterator

Symbol 不能使用 new 关键字构造调用（报错 TypeError）

Symbol 无法使用 new，因为 new 关键字调用会创建一个显式的 Symbol包装器对象，并非 Symbol值 —— MDN Symbol

可以使用 Symbol.for 创建全局共享的符号

Symbol 实际上并非私有属性，可以使用 Object.getOwnPropertySymbols(...) 访问到，但是可以用作私有属性或特殊属性来代替
